1. Problem Description

This website entails of my own gym idealogies and also ideas. Everything that you see comes from my own mind and also my heart. It strives to also help non gym goers understand the contents of what the gym lifestyle contains and what they can learn from it. It shows what they can learn of how it benefits them, different programs, benefits of the gym, different training methods in the gym and also a little bit about myself and my own gym journey. Furthermore, it has a gym related form and also gym related questions.

The archetypical user of this site may be somebody who has an interest in getting started in the gym but does not yet know where to start. They might have a previous or current interest in sport and wish to take their game to the next level by incorporating gym workouts. They may either be someone who has zero experience of any sport or any fitness related activities, however they may want to make a change in their life.

The problem that this site solves is it may offer some motivation for those who have either lost interest in the gym or those who are quite nervous or anxious to begin their gym journey. Those who wish to learn more about the gym may also have their problems solved for this site is very factual and everything stated is true.

3. Technology Selection and Site Architecture

The version of HTML that my website used was developed in HTML5 and the version of CSS that was used was CSS3. These are the two most recent versions that are available to the general public and so I felt like they would be the two best equipped versions to incorporate my websites needs. My plan for the site layout was to include the same header picture that matched the overall theme of the website as a whole. It would also include a navigation bar that would allow you to access all 7 of the web pages in the website. Below this will include a heading that shall be on every single page with another smaller heading below stating the page name. All pages will have this similar look and feel, but every pages contents will vary depending on the pages title. My plan for responsive design was to use media queries that would allow the web page to look and feel quite different on different devices and screen sizes.

We expect users to view the web page on any screen size as many are available to most users. This meant that it was vital that I included media queries that would allow the web page to look and feel as it should on each corresponding device. Media queries change the CSS styling of the page depending on the screen sizes which allow for easier use of the web pages depending on the device the user wishes to view the site on.

In order to keep my website cross browser compatible I ensured I kept my HTML markup simple along with my CSS styling and also my javascript. This is to ensure that the web pages are completely cross-browser compatible. I also made sure that my forms and also quiz were validated. I made sure to set my doctype to html. Last but not least I also made sure to link all my web pages to the javascript and css pages. This was to ensure all browsers shall allow the css and javascript along with the html.

6. Testing Plan

Explain in detail the strategies used to test the website. (eg. HTML validation, CSS validation). Is the site Cross Browser Compatible?

I used a HTML validator from W3C to test the website's html validation. This was done to make sure that my HTML markup had no errors and that it was not leading astray to the HTML standards needed and set by W3C. This was done to make sure that my HTML worked well with the web pages and ensured the website was working properly and as desired by me for the users.

The CSS validation was done the exact same way however, it was with my css file instead of my html file. This ensured that my CSS was up to standard and that it had no errors too.

A combination of the HTML and also CSS validation was to ensure there was no errors in either of my languages. Having no errors allows every single browser to run the website as normal without any issues. This means that the web pages are all cross-browser compatible. For example if I have an error in my markup my web page might run normally on google chrome but possibly not on firefox. This is why I made sure that I validated both of my languages to ensure there was no errors.

7. Site Evaluation

The first way that I evaluated my website was through the heuristic of user control and freedom. I ensured that no user will find themselves in places that they do not expect whilst browsing my website. I have each page labelled correctly along with the navigation bar too. I also have the page number of each page at the bottom of each page to ensure the user does not find themselves to be lost. Each page clearly shows where the user is and the navigation bar proves to be a quick and easy way of navigating through the pages.

The second way that I evaluated the website was through the use of the heuristic of consistency and standards. This means that the site layout on each page remained very alike to other pages on the web and also felt true to the other pages on my website. This ensured that I kept a good standard and also consistency throughout the entire website. I felt as though the website was designed well and any user should find the usability of the website to be fairly simple to navigate and find any of the content of the website.

The third way that I evaluated the website was through the use of the heuristic of match between system and the real world. I ensured that any text on my website did not use gym jargon. This was to ensure that each and every user could understand the meaning of anything stated in the website. It allowed them to be familiar with the meaning of every single thing in the website. All information about the gym and exercise was shown in the website to the users in a clear and easy to comprehend way.
